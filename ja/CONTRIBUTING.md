# 貢献に関するメモ

1. 新しい記事を提出する場合は、まずバグトラッカーを確認してください。まだ課題が登録されていなければ、作業を始める前にバグとして登録してください。役に立つ議論が起きる可能性があります。自分の時間を無駄にする必要はありません。

2. 英語で貢献してください（つたない英語でも**問題ありません**。メンテナが手助けしますし、**改善されます!**）。すべての Lisp コードが動くことを確認し、検証に使った具体的な Lisp 処理系を記してください。すべてのリンクが動くことも確認してください。

3. ライセンスを確認してください。あなたの作業は編集、コピー、変更される可能性があり、それを受け入れられる必要があります。また、著作権を持っていないコードをコピー＆ペーストしないでください。

4. 連絡手段としてバグを登録することをためらわないでください。そうしておくと、他の人がはるかに見つけやすくなります。
