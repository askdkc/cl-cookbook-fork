クックブックへの貢献ありがとうございます。以下のガイドラインに従ってください。一部は単なる慣習ですが、一部は epub 生成にとって重要です。

## 見出し

- （重要）章の最初の節は小見出しにしてください: `## title`（実際、epub での節名が章名になります）。

- 見出しは文のように書き、最初の単語だけを大文字にします。

- 関数を参照する場合は、Markdown の記法も使ってください（バッククォート）。

## リスト

- 番号付きの箇条書き（`1.`）は最初の階層でのみ使ってください。次の入れ子の階層では、通常の番号なしリストを使います。
- リストは最初の項目の前に空行を置く必要があります。


## コードの書式

- 関数は通常バッククォートで参照してください。Markdown の時代に大文字化する必要はありません。関数ではなく `function` と書いてください。

- 主流のスタイルに合わせてコードをインデントしてください。わからない場合は Emacs の `lisp-mode` がどうするかを確認してください。

## コードスニペット

- コードスニペットには `~~~lisp` を使ってください。

- スニペットは行頭から始める必要があります。`~~~lisp` のコードフェンスをインデントしないでください。

- コードスニペットの前後には空行が必要です。

```

Here's a snippet:

~~~lisp
(defun oh ())
~~~

This snippet...
```

- 例にとって非常に重要でない限り、"CL-USER> " のような Lisp のプロンプトは書かないでください。これは主にコピー＆ペーストしやすくするためです。

- スニペットの結果を示すには、スニペットが1行だった場合は同じ行に `;; => result` を使います。そうでない場合は次のようにできます。

~~~lisp
(print :abc)
;; :abc
~~~

結果が大きい場合は、コメントなしの別のコードブロックを使ってください。

- 行は 64 文字を超えないようにしてください。そうしないと PDF で切り詰められ、横スクロールバーが表示されます。


## EPUB の注意点

### 埋め込みコンテンツ

youtube 動画などの埋め込みコンテンツを乱用すべきではありません。

次のフラグで EPUB 生成から除外してください。

    <!-- epub-exclude-start -->
    <!-- epub-exclude-end -->

### 内部リンク（EPUB 用）

EPUB リーダーで内部リンクが動くように、Markdown のリンクを調整する必要があります。リンクを追加するには `fix-epub-links.sed` を参照するか、GitHub で声をかけてください。

## PDF の注意点

### 画像

現在は Markdown のソースを Typst へ変換してから PDF にしているため、HTML の `<img>` タグは処理されません。

`![](path/to/image.png)` の記法は問題なく動きますが、HTML タグは `max-width` などの CSS 属性を設定するのに便利です。

したがって、画像を追加するには次のようにしてください。

```txt
<img src="assets/sockets-lisp-chat.gif"
     width="470" height="247" alt="Chat app demo between two browser windows"/>

<!-- pdf-include-start
![](assets/sockets-lisp-chat.gif)
   pdf-include-end -->
```

最初の IMG は HTML 表示と EPUB でそのまま動きます。一方、HTML コメントの間にある2つ目は常に無視されますが、PDF をビルドするとき（`make pdf`）には存在するようにしています。
